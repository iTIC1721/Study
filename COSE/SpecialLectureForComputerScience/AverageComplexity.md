# 🅿️평균 복잡도와 P vs NP

어떤 문제가 NP-완전(NP-complete)이라고 해서, 그 문제가 항상 어려운 것은 아님
- 두 문제가 모두 NP-완전이더라도, 하나는 실제로 자주 나오는 입력에서는 빠르게 풀리고, 다른 하나는 그렇지 않을 수 있음  
-> NP 완전이다는 것은 **"최악의 경우"**에 그렇다는 이야기, 평균적으로 어렵다는 보장은 없음!

## Impagliazzo의 5가지 세계
러셀 임파리아조(Russell Impagliazzo)는 P vs NP 문제의 답에 따라 우리가 살 수 있는 세계를 5가지로 나누었음

### 1. Algorithmica (알고리스미카)
P = NP 인 세계 -> 검증이 빠르면 풀기도 빠른 세계
- 창의성이 필요했던 모든 작업(수학 증명, 공학 설계, 작곡, 그림 등)을 컴퓨터가 자동으로 수행 가능
- VLSI 설계를 휴리스틱 없이 최적으로 할 수 있음
- 프로그래밍이 "어떻게 할지"가 아니라 "무엇을 원하는지"만 명시하면 됨
- 단, 모든 암호 시스템이 붕괴됨 (공개키 암호화 불가능)

+ BPP = P라는 알려진 정리도 성립 (if P=NP)

AI와의 연결 (기술적 특이점?): 알고리스미카에서 AI는 자기 자신보다 더 스마트한 기계를 만들 수 있고, 이것이 기술적 특이점(Technological Singularity)의 개념과 연결

### 2. Heuristica (휴리스티카)
NP 문제가 최악의 경우엔 어렵지만, 어떤 효율적 샘플링 분포에 대해서도 **평균적으로는 쉬운 세계**
- 어려운 입력은 존재하지만, 그 입력을 찾는 것 자체가 어려움
- "마법의 알고리즘"이 거의 모든 NP 문제를 풀지만, 실패하는 입력이 있을 수 있음
- 실패 입력을 실제로 마주칠 일은 거의 없으므로, Algorithmica와 구별하기 어려움

예시 - VLSI 회로 최소화 문제 (Heuristica에서의 2단계 알고리즘):
1. 어떤 회로가 특정 함수를 올바르게 계산하는지 인식하는 알고리즘 (Co-NP 문제)
2. NP-완전 문제에 대한 알고리즘으로 최소 회로를 찾음

### 3. Pessiland (페시랜드)
어려운 평균-케이스 문제는 존재하지만 **일방향 함수(one-way function)는 없는 세계** (최악의 세계)

#### 일방향 함수란?
f(x)를 계산하기는 쉽지만 f(x)가 주어졌을 때, f(x') = f(x)인 x'를 찾기는 어려운 함수

페시랜드에서는 어려운 NP 문제 인스턴스를 생성하는 건 쉽지만, 그것을 암호학적으로 활용할 수 없음 → **암호화 불가능, 어렵기만 한 세계**

### 4. Minicrypt (미니크립트)
일방향 함수가 존재하지만 공개키 암호화는 불가능한 세계
- 정수 인수분해 같은 구조화된 NP 문제들은 다항 시간에 풀림
- 비밀키 암호화 등 일부 암호 응용은 가능
- 낯선 사람과 공개 채널에서 비밀 공유는 불가능

### 5. Cryptomania (크립토마니아)
대수(factoring)처럼 구조화된 문제가 평균적으로 지수적으로 어려운 세계
- 공개키 암호화가 가능
- 대부분의 연구자들이 우리가 실제로 사는 세계라고 믿음

이 세계에서:
- 일방향 함수로 어렵고 풀린 문제를 생성할 수 있음: x를 골라 y = f(x) 계산 → "f(x') = y인 x'를 찾아라"
- RSA 등 공개키 암호가 작동
- 단, 범용 알고리즘 없이 휴리스틱/근사/창의성으로 중요한 계산 문제들을 풀어야 함

#### RSA 암호
1. 두 소수 p, q 선택 (비밀)
2. r = p × q 계산 (공개)
3. (p-1)(q-1)과 서로소인 e 선택 → 암호화 키
4. d = e의 역원 mod (p-1)(q-1) → 복호화 키 (비밀)
5. r과 e는 공개, d는 비밀 유지

- 암호화: X가 Y의 공개키로 메시지 암호화 → Y만 자신의 비밀키로 복호화 가능
- 서명: X가 자신의 비밀키로 암호화 → Y가 X의 공개키로 복호화(진위 확인)

## AV-수학 (Algorithmically Verifiable Mathematics)
증명 검증 문제가 컴퓨터로 계산 가능한, 일관성 있고 충분히 강력한 형식 이론

임의의 텍스트 T가 주어졌을 때:
- YES → T가 증명임
- NO → T가 증명이 아님  
==> 이것이 컴퓨터로 판별 가능

<img width="545" height="107" alt="image" src="https://github.com/user-attachments/assets/ce05820f-2771-4289-a000-123397d13d46" />

### 괴델의 제1 불완전성 정리
건전한(sound) 형식 체계 F가 있을 때:  
"m은 K의 원소가 아니다"라는 것이 사실이지만, F 안에서 증명 불가능한 자연수 m이 존재한다.

==> **참인 명제라도 증명할 수 없는 것이 반드시 존재!!**

#### 형식 체계
공리에서 출발해 추론 규칙을 반복 적용하여 정리(theorem)를 생성하는 컴퓨터블 함수

**구성요소:**
1. 잘 형성된 공식(wff)의 결정가능한 집합
2. 공리(axiom)의 결정가능한 집합
3. 추론 규칙(inference rule)의 집합

K는 재귀적으로 열거 가능(recursively enumerable)하지만, 계산 가능(computable)하지는 않음

### AV-수학의 한계

AV-수학 안에서 증명할 수 없는 것들:
1. 빈 문자열에서 멈추지 않는 프로그램 P가 존재하며, 이 사실이 AV-수학에서 증명 불가능한 경우가 있다
2. 무한히 많은 TM A에 대해, A가 멈추지 않는다는 사실이 AV-수학에서 증명 불가능하다
3. 무한히 많은 TM M에 대해, M이 다항 시간에 작동하는지 여부를 AV-수학에서 증명할 수 없다
4. 무한히 많은 알고리즘에 대해, 그것이 SAT를 풀지 않는다는 것을 AV-수학에서 증명할 수 없다
5. P=NP이면, "X가 SAT를 다항 시간에 푼다"는 명제를 증명하거나 반증할 수 없는 알고리즘이 무한히 많다

## 라이스의 정리 (Rice's Theorem)
> F가 계산 가능한 부분 함수들의 비자명(non-trivial) 부분집합이라면,
> "어떤 프로그램이 F에 속하는 함수를 계산하는지" 결정하는 알고리즘은 존재하지 않는다.

==> **프로그램의 행동(semantic property)에 관한 비자명한 성질은 모두 결정 불가능!!**

- 구문적 성질(syntactic): 프로그램에 if-then-else가 있는가? → **결정 가능**
- 의미적 성질(semantic): 프로그램이 모든 입력에서 멈추는가? 제곱 함수를 계산하는가? → **결정 불가능**

## 오라클과 P vs NP
오라클 B를 이용하면 P^B ≠ NP^B인 경우를 구성할 수 있음

### Finicky Oracle 문제
입력: n (단항 표현)  
질문: B가 "yes"라고 답하는 길이 n인 문자열 wₙ이 존재하는가?

==> 이 문제는 NP^B에 속함 (증인 wₙ을 B에 물어보면 검증 가능), 하지만 P^B에는 속하지 않음  
이유: 다항 시간 프로그램 Π는 길이 n의 문자열 중 poly(n)개만 물어볼 수 있는데, 2ⁿ개 중에서 압도적으로 대부분에 B는 "no"라고 대답하므로 wₙ의 존재 여부를 알 수 없음

==> P vs NP 문제를 단순한 대각화(diagonalization) 기법으로는 해결할 수 없음을 시사
